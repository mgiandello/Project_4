Can we create a model that can be used to predict a range of box office earnings and imdb scores for future debuts?

Goal


Big-budget movies carry a large financial risk. In order to help make decisions on a release strategy, we started an application to predict the box office success and IMDb ratings of a film.
We gathered data on historical movie releases from the past 42 years to create an algorithm and predict a range of box office earnings for future debuts.

Unfortunelty, we are still in our model is still in Beta testing.


Approach


We followed a traditional extract, transform, and load process to create a database hosted on Heroku. The data we collected includes budget, runtime, genres, directors, actors, meta-scores, IMDb ratings, and more.

We gathered data regarding releases over the past 42 years (1980-present) as we wanted to collect the most data possible that was avaioble to us. The target movies was determined to be upcoming films releasing over the next several months in 2022.


Process



Extraction:


Movie titles and budget data was gathered from thenumbers.com
The majority of historical data was called from the OMDb API and Kaggle.
The data on upcoming movies was gathered from IMDb.
Web scrapers were used to gather non-API data.


Transformation:


Data was cleaned using Jupyter notebook scripts.
All of the data gathered from OMDb was stored as objects, so all datatypes were converted appropriately to load into the database.
Movie titles for the upcoming released films scraped from IMDb included year of release in the string. This attribute was removed via string split.
Omitted data included: language, plot, and country (We kept to top Domestic Films only).
Release date information was converted to Month.
Features gathered from separate sources were all merged to a single dataframe on the movie title.


Loading:


CSVs containing the final cleaned data were imported into Heroku via Jupyter notebook scripts connecting to PostgreSQL.
pgAdmin was our preferred way of storing our data locally.


Machine Learning Models:


Data was called from our database and used to create an Random Forest Classifer and Logistic Regression Models.
Dummy columns were created for genre and movie rating.
The historical dataset was split into testing and training sets.
Once optimized, the model was exported, allowing it to be accessible to our application.


Presenting:


Our model was deployed using a Flask application, built with Python, which applies our model to data describing upcoming movies.
A website, hosted through Heroku, creates an interface in which a user is able to select an upcoming film to view our prediction for that filmâs Box Office sales and IMDb rating.

Challenges




In data collection:


We used the available data we had from web scrapping and Kaggle, but our models were not accurate with the amount of features we had. Important features like a movie's budget, which would be helpful to predict the Box Office outcomes, is mostly data is behind a ignificant paywall. Therefore, this has lead to an issue in trying to predict Box Office and IMDb ratings. More features are needed to make our accuarcy better.


In data cleaning:


As we moved into the data cleaning portion of the project, the shape of our data began to change. We encountered noisy, inconsistent, and missing data. For these reasons we elected to not include the ceratin features like language, plot, and country. After dropping irrelevant columns, changing the release date to a month column, creating dummy columns for genre and movie rating, and scaling the data, we were ready to test out some models.


In modeling:


Initially, we tried multiple models and target data binning methods to see which ones would yield the best result. Linear regression and SVM models didnât seem suitable for the dataset that we assembled, so we moved on to random forests, and logstic regression. These 2 models yielded different accuracy scores:

Random Forest Model: Box Office

Training Score: 1.0
Testing Score: 0.223

These testing scores are very low, in which predicting the Box Office revenue was very inaccurate from Prediction vs. Actual just on the Histroical Movie Data alone. This is when we decided Random Forest was not the best choice.

Logistic Regression Model: Box Office

Training Score: 0.698
Testing Score: 0.716

These testing scores are better than Random Forest, but still low in the low 70% accurarcy. Therefore, Box Office revenue was still inaccurate from Prediction vs. Actual on Historical Move Data and Upcoming Movie Data.

Logistic Regression Model: IMDb Ratings

Training Score: 0.702
Testing Score: 0.712

These testing scores are better than Random Forest as well, but still low in the low 70% accurarcy. IMDB ratings was better, but still inaccurate from Prediction vs. Actual on Historical Move Data and Upcoming Movie Data.



In the end, Logsitc Regression seems like the better mmodel to use between the two machine learning models. However, we still need more features to get a higher accuracry score use Logsitc Regression properly.
